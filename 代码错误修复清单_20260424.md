### 【P0-1】JWT Token 中 `tenantId` 为可选，但大量路由直接使用不报错

**文件**: `backend/src/common/utils/jwt.ts` 第 27 行

**当前代码**:
```typescript
export interface JwtPayload {
  id: string;
  type: 'developer' | 'user';
  tenantId?: string;   // ← 这里用了 ?，说明可以为 undefined
  departmentId?: string;
  role?: string;
  // ...
}
```

**问题**: 开发者登录后 `tenantId` 为 `undefined`，但下游路由直接用 `req.user.tenantId` 做查询条件，不报错却查不到数据。

**修复方案**: 在 `authenticate` 中间件中强制补全 tenantId：

在 `backend/src/common/middleware/auth.ts` 中的 `verifyToken` 之后、调用 `next()` 之前插入：

```typescript
// 在 verifyToken(token) 之后、req.user = decoded 之后：
if (!decoded.tenantId && decoded.type === 'user') {
  const { prisma } = await import('../utils/prisma');
  const user = await prisma.user.findUnique({ where: { id: decoded.id } });
  if (user) {
    decoded.tenantId = user.tenantId;
    decoded.departmentId = user.departmentId;
    decoded.role = user.role.name;
  }
}
```

```typescript
export interface AuthenticatedRequest extends Request {
```

### 【P0-4】`Role.name` 是全局唯一，导致多租户角色冲突

**文件**: `backend/prisma/schema.prisma` 第 303 行

**当前代码**:
```prisma
model Role {
  id          String   @id @default(cuid())
  tenantId    String
  tenant      Tenant   @relation(fields: [tenantId], references: [id], onDelete: Cascade)
  name        String   @unique  // ← 全局唯一，租户A的 admin 和租户B的 admin 会冲突
  // ...
}
```

**修复方案**:
```prisma
name        String
@@unique([tenantId, name])  // 改为租户内唯一
```

⚠️ **注意**: 修改 Prisma schema 后需要运行 `npx prisma migrate dev` 并重新生成迁移文件，原有数据需要迁移。

### 【P1-3】SQLite 使用了 `@db.Decimal`（SQLite 不支持）

**文件**: `backend/prisma/schema.prisma` 多处

**涉及字段位置**（约 30+ 处）:

| 行号 | 字段 | 行号 | 字段 |
|------|------|------|------|
| 123 | `pricePerMonth` | 782 | `amount`（产值） |
| 124 | `pricePerExtraUser` | 783 | `payableRatio` |
| 144 | `amount`（进度款） | 801 | `totalAmount`（分包） |
| 168 | `totalAmount`（合同） | 837 | `monthlySalary` |
| 193 | `amount`（进度款2） | 838 | `dailySalary` |
| 458 | `unitPrice` | 839 | `workerDailySalary` |
| 488 | `quantity`（库存） | 840 | `socialInsurance` |
| 489 | `outQuantity` | 897 | `attendanceDays` |
| 534-537 | `deliveryQty/receivedQty/actualQty/unitPrice` | 898-905 | `dailyWage/basePayable/absentDeduction/overtimePay/socialInsuranceDeduction/totalPayable/totalPaid/arrearsAmount` |

**问题**: SQLite 没有 `Decimal` 类型，Prisma 会将其映射为 `TEXT`，导致金额精度丢失。

**修复方案**: 全部改为 `Float`：

```bash
# 在 schema.prisma 目录下执行全局替换
# 将所有不带参数的 Decimal 替换为 Float
# 注意：@db.Decimal 是无效注解（仅 PostgreSQL/MySQL 有效）
```

⚠️ **注意**: 已有数据需要迁移，金额字段从文本转回数字。

**当前代码**:

```typescript
// 兼容读取（优先新key）
const token = localStorage.getItem('zlt_token')
             || localStorage.getItem('wms_token')
             || localStorage.getItem('token');
```

### 【P2-2】`PaymentRecord.personnelId` 可选，导致风控漏检

**文件**: `backend/prisma/schema.prisma` 第 938 行附近

**当前代码**:
```prisma
model PaymentRecord {
  personnelId     String?  // ← 可选，发放记录可以不关联人员
  // ...
}
```

**问题**: 风控异常检测依赖 `personnelId`，如果为 `null` 则跳过人脸/身份证比对。

**修复方案**: 如果业务上允许无人员关联的发放，保持现状但补充说明；否则改为必填：

```prisma
personnelId     String  // 移除 ?
```

### 【P2-3】`Subscription` 每个租户只能有一条订阅记录

**文件**: `backend/prisma/schema.prisma` 第 120 行

**当前代码**:
```prisma
model Subscription {
  tenantId   String @unique  // 每个租户只能有一条订阅
  // ...
}
```

**问题**: 无法用多条订阅记录表达多个计费周期（季度/月度/年付）。

**修复方案**: 增加周期字段：

```prisma
model Subscription {
  tenantId           String @unique
  currentPeriodStart DateTime
  currentPeriodEnd   DateTime
  // 方案B：独立计费周期表（更灵活，推荐）
}
```

### 【P2-4】API 错误响应格式不统一

**文件**: 多处路由文件（`modules/wms/routes.ts`、`modules/labor/routes.ts` 等）

**问题**: 有的返回 `{ success: false, error: 'CODE', message: '...' }`，有的缺 `error` 字段，有的缺 `message` 字段。

**修复方案**: 在 `common/utils/` 下新增 `response.ts`：

```typescript
// common/utils/response.ts
import { Response } from 'express';
import { ApiResponse } from '../types';

export function successResp<T>(res: Response, data: T, message?: string): void {
  res.json({ success: true, data, ...(message && { message }) });
}

export function errorResp(res: Response, status: number, code: string, message: string): void {
  res.status(status).json({ success: false, error: code, message });
}
```

```typescript
  localStorage.removeItem('zlt_token');

    window.location.href = '/login';
  }
```

**建议**: 增加防重复判断：
