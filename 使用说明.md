## 二、本地开发

### 2.1 安装依赖

```bash
# 进入项目目录
cd 资料通工程管理系统

# 安装后端依赖
cd backend && npm install && cd ..

# 安装前端依赖
cd frontend && npm install && cd ..
```

### 2.2 数据库初始化

```bash
# 确保 PostgreSQL 已启动并创建了数据库
# 数据库名：zlt_management
# 用户名：zlt_user
# 密码：Zlt_password_2024

cd backend

# 生成 Prisma Client
npx prisma generate

# 执行数据库迁移
npx prisma migrate dev

# 导入种子数据
npm run prisma:seed
```

### 2.3 启动开发服务

```bash
# 终端1：启动后端（端口 4001）
cd backend
npm run dev

# 终端2：启动前端（端口 5173）
cd frontend
npm run dev
```

访问 http://localhost:5173 即可使用系统。

### 2.4 Docker 一键启动

```bash
# 在项目根目录执行
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down

# 停止并清除数据
docker-compose down -v
```

## 三、配置文件说明

### 3.1 后端环境变量（backend/.env）

| 变量名 | 说明 | 默认值 |
|--------|------|--------|
| PORT | 后端服务端口 | 4001 |
| NODE_ENV | 运行环境 | development |
| DATABASE_URL | PostgreSQL 连接串 | postgresql://zlt_user:Zlt_password_2024@localhost:5432/zlt_management |
| JWT_SECRET | JWT 签名密钥 | **必须修改** |
| JWT_EXPIRES_IN | Token 有效期 | 7d |
| UPLOAD_DIR | 上传文件目录 | ./uploads |
| MAX_FILE_SIZE | 最大文件大小（字节） | 52428800 (50MB) |
| FRONTEND_URL | 前端地址（CORS） | http://localhost:5173 |

### 3.2 AI/OCR 配置

AI 和 OCR 的配置优先从数据库读取（开发者后台配置），如果数据库没有配置则使用环境变量。

| 变量名 | 说明 |
|--------|------|
| MINIMAX_API_KEY | MiniMax 大模型 API Key |
| MINIMAX_GROUP_ID | MiniMax 组 ID |
| BAIDU_OCR_API_KEY | 百度 OCR API Key |
| BAIDU_OCR_SECRET_KEY | 百度 OCR Secret Key |
| TENCENT_OCR_SECRET_ID | 腾讯云 OCR Secret ID |
| TENCENT_OCR_SECRET_KEY | 腾讯云 OCR Secret Key |

### 3.3 前端环境变量

前端通过 Vite 代理访问后端 API，无需额外配置环境变量。

生产构建时，API 地址通过 Nginx 反向代理处理。

## 五、生产部署

### 5.1 服务器部署步骤

```bash
# 1. 克隆代码
git clone <repository-url>
cd 资料通工程管理系统

# 2. 配置环境变量
cp backend/.env.example backend/.env
# 编辑 .env 填入生产环境配置

# 3. Docker 部署
docker-compose up -d --build

# 4. 初始化数据库
docker-compose exec backend npx prisma migrate deploy
docker-compose exec backend npm run prisma:seed
```

### 5.2 Nginx 配置（宿主机）

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;

    location / {
        proxy_pass http://localhost:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 5.3 数据库备份

```bash
# 手动备份
docker exec zlt_postgres pg_dump -U zlt_user zlt_management > backup_$(date +%Y%m%d).sql

# 恢复备份
cat backup_20260424.sql | docker exec -i zlt_postgres psql -U zlt_user zlt_management
```
