## 一、项目背景

### 1.1 起源

系统起源于两个独立开发的子系统：
1. **物资管理系统（WMS）**：bk_wms_20260421_121929，管理建筑物资的入库、出库、退库、调拨等
2. **劳资管理系统（Labor）**：bk_labor_20260421_121929，管理人员、考勤、工资核算、风控等

两个系统采用相同技术栈但独立部署，各自有独立数据库。随着业务发展，需要将它们集成到统一平台。

### 1.2 集成目标

- 统一数据库（从 2 个数据库合并为 1 个）
- 统一认证和权限体系
- 统一前端界面（模块化导航）
- 新增合同-项目部-子项目三级组织结构
- 新增 SaaS 订阅计费功能
- 支持多端访问

### 2.3 多租户架构

采用 **Developer → Tenant → Department** 三级模型：
- **Developer**：平台开发者/超级管理员，管理所有租户
- **Tenant**：施工企业（租户），拥有独立的数据空间
- **Department**：项目部，数据隔离的核心单元

数据隔离通过 Prisma 中间件实现，所有查询自动注入 `tenantId` 和 `departmentId` 条件。

### 2.4 权限体系

采用 **RBAC + 数据范围** 双重控制：
- **RBAC**：基于动态 Role + Permission 表，控制功能权限
- **数据范围**：ALL（全企业）/ DEPARTMENTS（授权列表）/ OWN_DEPARTMENT（所属项目部）
