## 一、系统定位

资料通工程管理系统是一款面向**建筑施工企业**的**多租户 SaaS 综合管理平台**，集成物资管理（WMS）和劳资管理两大核心模块，目标是实现工程项目的数字化管理。

---

## 二、角色与权限需求

### 三种用户层级

| 角色 | 职责 |
|------|------|
| **平台开发者（超级管理员）** | 管理所有租户、平台配置、系统监控 |
| **企业管理员** | 租户内最高权限，拥有全部功能 |
| **企业普通用户** | 按角色分配功能和数据权限 |

### 8 种业务角色

超级管理员、老板、管理员、物资员、劳资员、财务、出纳、项目出纳

### 核心权限需求

- 老板/财务总监/超级管理员拥有所有合同及子目录的劳资、物资、合同权限
- 项目经理可拥有多个项目的劳资、物资权限
- 项目劳资员可拥有多个项目的劳资权限
- 项目材料员可拥有多个项目的物资权限
- 可按层级批量给权限（总部层级 = 所有合同权限，项目层级 = 可选多个项目部）
- 所有权限含读取、修改、删除
- 所有操作有日志，只有超级管理员能删除日志

## 三、物资管理（WMS）需求

1. **物资目录**：编码、名称、规格、单位、单价、分类
2. **送货单 OCR**：上传图片/PDF → 百度 OCR 识别 → AI 结构化提取 → 自动匹配系统数据
3. **入库管理**：手动录入、Excel 批量导入、送货单转入库（三级联动：合同→项目部→项目名称）
4. **库存查询**：多维度筛选、待分配库存标识
5. **出库管理**：选择项目部→项目名称→勾选库存→填写数量领用班组、自动生成 PDF 出库单
6. **退库管理**：从出库记录勾选退库、Excel 批量退库、自动恢复库存
7. **物资调拨**：A→B 项目部/项目名称借调、借调归还
8. **班组台账**：按班组查询领料明细、导出 Excel
9. **供应商管理**、**班组管理**

---

## 四、劳资管理需求

1. **人员管理**：基本信息、薪资信息、入职/离职/复职流程、自动检测（未成年、超龄、身份证重复）
2. **考勤管理**：每日考勤（出勤/半勤/缺勤 + 加班）、批量录入、每月唯一考勤记录
3. **工资核算**：批量自动化核算、日薪制/月薪制、手动调整、欠薪追踪、考勤变更自动标记重算
4. **工资发放**：录入发放记录、支持非在册收款人、AI 辅助匹配、欠薪抵扣
5. **分包商管理**：公司/个人类型、基本信息
6. **分包合同管理**：关联总包合同 + 分包商、金额管理
7. **产值管理**：按月录入产值、分包进度款支付
8. **风控异常**：黄/红两级自动标记、异常处理（需填原因）、风控统计面板
9. **报表导出**：多月合并 Excel（4 Sheet 综合报表）
10. **附件管理**：文件上传（多文件、50MB 上限）

## 五、合同与项目管理需求

### 三级组织结构
```
合同 → 项目部 → 项目名称/子项目
```
- 单个合同可设多个项目部
- 单个项目部可有多个项目名称
- 项目部之间数据不互通
- 总部管理员/财务可对所有项目部选择权限

### 合同管理
- 类型：采购合同 / 承包合同
- 基本功能：名称、编号、金额、供应商、起止日期
- 进度款管理：记录每笔收款、合同金额 vs 累计收款对比
- 支持软删除

- API 密钥管理
- 系统公告（info/warning/maintenance）
- 安全策略
- 系统监控
- 用量统计、操作日志
- 回收站（数据恢复/永久删除）
- 企业视角切换（临时模拟企业用户身份）

## 十、人脸识别 APP 需求（待开发）

- APP 名称：资料通劳资系统
- 功能：人脸录入 + 打卡两个按钮
- 权限：项目部劳资员、班组负责人

- 身份证自动识别，与 APP 信息不一致时纠正
- 先开发安卓版
- APP 要有定位功能
