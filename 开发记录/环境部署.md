## 二、本地开发环境部署

### 2.1 一键启动（推荐）

```bash
cd "/Users/Namson/Documents/Claude code/资料通工程管理系统"
chmod +x start-dev.sh
./start-dev.sh
```

该脚本自动完成：
1. 检查 Node.js 版本
2. 启动 PostgreSQL Docker 容器（如未运行）
3. 安装后端 + 前端依赖
4. 生成 Prisma Client
5. 同步数据库结构
6. 导入种子数据
7. 创建上传目录

### 2.2 手动启动

```bash
# 1. 安装依赖
cd backend && npm install && cd ..
cd frontend && npm install && cd ..

# 2. 初始化数据库
cd backend
npx prisma generate
npx prisma db push
npm run prisma:seed
cd ..

# 3. 启动后端（端口 4001）
cd backend && npm run dev &

# 4. 启动前端（端口 5173）
cd frontend && npm run dev &
```

访问 http://localhost:5173

### 2.3 环境变量配置

在 `backend/` 下创建 `.env` 文件，参考 `REQUIREMENTS.md` 中的环境变量清单：

```env
PORT=4001
NODE_ENV=development
DATABASE_URL=postgresql://zlt_user:<数据库密码>@localhost:5432/zlt_management
JWT_SECRET=<生产环境必须修改为强随机字符串>
JWT_EXPIRES_IN=7d
UPLOAD_DIR=./uploads
MAX_FILE_SIZE=52428800
FRONTEND_URL=http://localhost:5173
```

## 三、Docker 生产部署

### 3.1 架构

```
┌─────────────┐   ┌─────────────┐   ┌──────────────┐   ┌──────────────┐
│   Nginx     │   │   Frontend  │   │   Backend    │   │  PostgreSQL  │
│  (宿主机)   │→  │  (容器:80)  │→  │ (容器:4001) │→  │ (容器:5432)  │
└─────────────┘   └─────────────┘   └──────────────┘   └──────────────┘
                                                               │
                                                        ┌──────────────┐
                                                        │    Redis     │
                                                        │ (容器:6379)  │
                                                        └──────────────┘
```

### 3.2 部署步骤

```bash
# 1. 进入项目目录
cd "/Users/Namson/Documents/Claude code/资料通工程管理系统"

# 2. 配置生产环境变量
cp backend/.env.example backend/.env
vim backend/.env  # 修改 JWT_SECRET、数据库密码等

# 3. 构建并启动所有服务
docker-compose up -d --build

# 4. 查看日志
docker-compose logs -f backend

# 5. 查看运行状态
docker-compose ps
```

### 3.3 服务说明

| 服务 | 容器名 | 端口 | 说明 |
|------|--------|------|------|
| postgres | zlt_postgres | 5432 | PostgreSQL 15 Alpine |
| redis | zlt_redis | 6379 | Redis 7 Alpine |
| backend | zlt_backend | 4001 | Express 后端 API |
| frontend | zlt_frontend | 3000→80 | Nginx 静态文件服务 |

### 3.4 常用运维命令

```bash
# 查看所有容器状态
docker-compose ps

# 查看后端日志
docker-compose logs -f backend

# 重启单个服务
docker-compose restart backend

# 停止所有服务
docker-compose down

# 停止并清除数据（危险！）
docker-compose down -v

# 进入后端容器
docker-compose exec backend sh
```

## 四、数据库管理

### 4.1 Prisma 命令

```bash
cd backend

# 生成 Prisma Client（schema 变更后必须执行）
npx prisma generate

# 查看数据库（可视化界面，端口 5555）
npx prisma studio

# 创建迁移（开发环境）
npx prisma migrate dev --name <迁移名称>

# 应用迁移（生产环境）
npx prisma migrate deploy

# 同步 schema 到数据库（不生成迁移文件，开发环境快速迭代用）
npx prisma db push

# 导入种子数据
npm run prisma:seed
```

### 4.2 数据库备份与恢复

```bash
# 备份 PostgreSQL
docker exec zlt_postgres pg_dump -U zlt_user zlt_management > backup_$(date +%Y%m%d).sql

# 恢复 PostgreSQL
cat backup_20260424.sql | docker exec -i zlt_postgres psql -U zlt_user zlt_management
```

## 八、OCR 功能依赖

OCR 送货单识别功能需要以下额外依赖，需在宿主机安装：

```bash
# macOS
brew install ghostscript

# Ubuntu/Debian
sudo apt-get install ghostscript python3-pil

# CentOS/RHEL
sudo yum install ghostscript python3-pillow
```

此外还需在开发者后台配置：
- **百度 OCR**：API Key + Secret Key
- **AI 大模型**：MiniMax 或 OpenAI 的 API Key
